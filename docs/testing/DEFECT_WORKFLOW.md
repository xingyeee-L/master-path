# 缺陷跟踪流程

## 1. 缺陷分类
- 功能缺陷：需求不符合、逻辑错误、边界条件失败。
- 体验缺陷：交互不符合预期、动画/反馈缺失、视觉不一致。
- 性能缺陷：卡顿、内存持续增长、首屏过慢。
- 安全缺陷：依赖漏洞、敏感信息泄漏、配置不当。
- 兼容缺陷：特定浏览器/平台异常。

## 2. 优先级定义
- P0：阻塞主流程/数据丢失/安全高危，必须立即修复。
- P1：核心功能错误或大量用户受影响，尽快修复。
- P2：次要功能问题或局部影响，计划内修复。
- P3：低影响体验问题，迭代优化。

## 3. 缺陷记录字段
- 标题：模块 + 现象 + 版本
- 环境：OS/浏览器/容器版本、分辨率、语言
- 复现步骤：最小可复现步骤
- 实际结果 / 期望结果
- 影响面：影响用户比例/关键路径
- 附件：截图/录屏/日志
- 优先级：P0-P3
- 根因分析：代码位置、触发条件、设计缺陷、回归风险
- 修复方案：改动点、测试点、回归用例

## 4. 流程状态
- New -> Triaged -> In Progress -> Fixed -> Verified -> Closed
- Reopen：验证失败或回归

## 5. 质量门禁
- 修复合并前必须通过：typecheck、lint、unit/integration tests
- 影响主流程的缺陷必须追加对应自动化用例
